# Security Program Charter
Burn and Churn Coffee

---

## 1. Purpose
The purpose of the Burn and Churn Coffee Security Program is to establish governance, risk management, and security practices that protect company systems, customer payment information, and business operations.

The program provides a structured approach for managing cybersecurity risks in a small retail environment that processes payment card transactions and relies on cloud-based business services.

This charter defines the scope, responsibilities, and guiding principles of the organization's security program.

--- 

## 2. Security Program Objectives
The Security program aims to:
  - Protect customer payment information during transactions
  - Safeguard business systems and data
  - Reduce cybersecurity risk affecting store operations
  - Provide clear security policies and responsibilities
  - Detect and respond to security incidents effectively
  - Align with recognized security frameworks and industry practices

---

## 3. Program Scope
The security program applies to all technology systems used by Burn and Churn Coffee, including:
  - Toast POS payment processing systems
  - Google Workspace email and collaboration services
  - Company-issues tablets used for store operations
  - Store network infrastructure (router, firewall, Wi-Fi access points)
  - Administrative access to POS and cloud systems

The program applies to all individuals interacting with company systems, including:
  - Business Owner
  - Store Manager
  - Shift Leaders
  - Employees
  - Authorized third-party service providers

----

## 4. Security Governance Structure
Security responsibilities are distributed according to the organization's size and structure.
### Business Owner
The **Business Owner** is responsible for overall cybersecurity governance, including:
  - Approving security policies
  - Reviewing cybersecurity risks
  - Authorizing incident response actions
  - Coordination with vendors and service providers
  - Ensuring compliance with applicable security practices
### Store Manager
The Store Manager assists with operational security responsibilities, including:
  - Supporting incident response coordination
  - Ensuring employees follow security policies
  - Monitoring operational systems for suspicious activity
### Employees
Employees are responsible for:
  - Following security policies and acceptable use requirements
  - Protecting company systems and credentials
  - Reporting suspicious activity or potential security incidents

---

## 5. Security Program Components
The Burn and Churn Coffee security program includes several key governance components.
### Policies
Security policies define expectations for system us and security practices.

Examples include:
  - Access Management Policy
  - Acceptable Use Policy
  - Incident Response Policy
  - Payment Card Data Protection Policy
### Risk Management
Cybersecurity risks are tracked and evaluated using the organization's Risk Register.

Risks are periodically reviewed to determine mitigation strategies.
### Control Management
Security controls are documented in the **Control Matrix**, which aligns controls with recognized security frameworks.
### Incident Response
Security incidents are managed in accordance with the **Incident Response Policy** and associated playbooks.
### Evidence and Documentation
Evidence supporting security controls in documented in the **Evidence Matrix** and associated artifacts.

---

## 6. Framework Alignment
The Burn and Churn Coffee security program aligns primarily with **NIST SP 800-53 control families**.

Examples include:
  - Access Control (AC)
  - Identification and Authentication (IA)
  - Incident Response (IR)
  - Audit and Accountability (AU)
  - Configuration Management (CM)

Where applicable, the program also incorporates security practices relevant to **PCI DSS** due to the organization's use of payment card processing systems.

---

## 7. Continuous Improvement
The security program is reviewed periodically to ensure it remains effective and appropriate for the organization's operational environment.

Reviews may occur:
  - Annually
  - Following major system changes
  - After significant security incidents
  - When new cybersecurity risks are identified

Improvements may include policy updates, new controls, or additional employee training.
