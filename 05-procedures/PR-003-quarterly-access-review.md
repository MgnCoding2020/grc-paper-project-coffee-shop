# Quarterly Access Review Procedure
Burn and Churn Coffee

## 1. Purpose
The purpose of this procedure is to ensure that user access to company systems and data is periodically reviewed to confirm that access remains appropriate for each user's job responsibilities.

Regular access reviews help reduce the risk of unauthorized access, excessive permissions, and lingering accounts that could expose company systems or sensitive information.

-------------

## 2. Scope
This procedure applies to all systems and services used by Burn and Churn Coffee that require user authentication or role-based permissions.

Systems included in the review process include, but are not limited to:
  - Google Workspace (email, Drive, administrative roles)
  - Toast POS system
  - xtraCHEF inventory and supplier management platform
  - QuickBooks Online accounting platform
  - Store networking systems and administrative interfaces

---------

## 3. Roles and Responsibilities
### 3.1 Business Owner / Management
  - Reviews and approves access review results
  - Confirms that employee access aligns with job responsibilities
  - Approves removal or modification of unnecessary access
### 3.2 System Administrator
  - Generates or collects user access lists from applicable systems
  - Provides access reports to management for review
  - Implements access changes identified during the review
### 3.3 Supervisors / Managers
  - Verify that employees under their supervision have appropriate access
  - Identify accounts that should be modified or removed

----------

## 4. Review Frequency
Access reviews must be conducted quarterly.

Suggested review periods:
  - Q1 – January to March
  - Q2 – April to June
  - Q3 – July to September
  - Q4 – October to December

The review should occur near the end of each quarter.

-----------

## 5. Access Review Process
### Step 1: Generate Access Reports
The system administrator gathers current user access lists from applicable systems.

Examples include:
  - Google Workspace user and admin role lists
  - Toast POS employee account permissions
  - xtraCHEF system user access
  - QuickBooks Online user access list

### Step 2: Manager Review
Managers review the user lists to confirm:
  - The user is still employed by the organization
  - Access aligns with the user's job responsibilities
  - No excessive or unnecessary privileges exist

Managers should identify any accounts that require:
  - Access removal
  - Permission reduction
  - Role modification

### Step 3: Access Updates
The system administrator implements any required access changes identified during the review.

Examples include:
  - Removing accounts for former employees
  - Adjusting permissions for role changes
  - Removing administrative privileges that are no longer required

### Step 4: Documentation
The results of the access review must be documented.

Documentation should include:
  - Date of review
  - Systems reviewed
  - Reviewer name
  - Summary of findings
  - Actions taken

This documentation serves as evidence that periodic access reviews are performed.

-----

## 6. Handling Inactive Accounts
If accounts are identified as belonging to former employees or inactive users, the system administrator must disable or remove them promptly.

Inactive accounts should not remain enabled unless a documented business reason exists.

----

## 7. Record Retention
Access review documentation should be retained for at least one year for internal review and audit purposes.

----

## 8. Procedure Review
This procedure should be reviewed annually or when significant changes occur to company systems or access control practices.
