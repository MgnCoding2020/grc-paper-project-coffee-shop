# Evidence Examples & Templates

This directory holds **blank templates and example artifacts** that support the controls
tracked in the [evidence matrix](../evidence-matrix.md). They are intended to be filled in
during periodic reviews, incidents, and vendor assessments, then stored as completed
evidence under `../artifacts/`.

| Template | Supports Control | Evidence ID | Purpose |
|----------|------------------|-------------|---------|
| [access-review-checklist.md](access-review-checklist.md) | AC-2, AC-6 | E-01, E-03 | Standardize quarterly user/privileged access reviews |
| [incident-report-template.md](incident-report-template.md) | IR (AU-6) | E-04 | Capture a consistent record for each incident |
| [pos-device-inspection-checklist.md](pos-device-inspection-checklist.md) | PE-3 | E-07 | Physical/tamper inspection of POS terminals (PCI Req. 9.5.1) |
| [security-training-acknowledgement-form.md](security-training-acknowledgement-form.md) | AT-2 | E-06 | Record employee completion of security awareness training |
| [vendor-security-documentation-summary.md](vendor-security-documentation-summary.md) | SA-9 | E-09 | Summarize security documentation collected from vendors |

> These are simulation artifacts for educational and portfolio purposes only.
